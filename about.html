// The Problem

Why WhiteNet Exists

Critical digital infrastructure lacks identity-level accountability. WhiteNet changes that.

🔴 THE PROBLEM

Today's internet was built for connectivity, not accountability. Anyone can connect, send packets, and operate services — often with little or no verified identity. This design flaw enables:

  • 🎭 Identity Spoofing — Actors impersonate legitimate services
  • 🕵️ Untraceable Attacks — No binding between identity and network address
  • ⚡ DNS Poisoning — Name resolution can be hijacked without verification
  • 🔓 Trust Assumptions — Systems trust connections by default, violating Zero Trust
  • 📡 Certificate Fraud — No lifecycle management for identity credentials

✅ THE WHITENET SOLUTION

WhiteNet creates an identity-first network layer where every participant must hold a CA-signed cryptographic certificate bound to a unique IPv6 address before any communication is allowed.

CORE PRINCIPLE

"No identity → No access.
Tampered identity → Blocked."

// Trust Flow

Identity Lifecycle & Trust Decision

From certificate issuance to real-time trust assessment — every step is cryptographically verifiable

01

CA Key Generation

RSA-2048 CA keypair is generated. The private key signs all certificates; the public key verifies them. ca_private.pem + ca_public.pem

02

Certificate Issuance

User identity gets a unique cert_id, RSA-2048 key pair, and CA signature. Certificate includes valid_until expiry timestamp (24h by default).

03

IPv6 Binding

Certificate is bound to a unique, randomly-generated IPv6 address. DNS record {user_id}.whitenet.local → IPv6 is created automatically.

04

Trust Verification

Before any communication, the node's certificate signature is re-verified against the CA public key. Tampered certificates are immediately detected and blocked.

05

Posture Assessment

7-factor trust scorecard produces a TRUSTED / WARNING / BLOCKED verdict with a 0–100 score. Checks: registry, certificate, revocation, expiry, DNS identity, audit chain, recent signals.

06

Secure Communication

TLS 1.3 handshake → cipher negotiation → VPN tunnel establishment. Only verified, non-revoked nodes can participate. All actions logged to the signed audit chain.

07

Lifecycle Management

Certificates can be revoked (blocked from network) or renewed (fresh certificate, same IPv6 binding). Governance proposals manage policy changes through community voting.

// Features

v3.0.0 Feature Set

Every feature enforces the identity-first, Zero Trust principle

🔐

PKI Certificate Authority

RSA-2048 CA issues signed identity certificates with unique cert_id, expiry timestamps, and revocation support.

🌐

IPv6 Identity Binding

Each certificate is cryptographically bound to a unique IPv6 address with automatic DNS record creation.

🛡️

7-Factor Trust Assessment

TRUSTED / WARNING / BLOCKED verdicts with 0-100 scoring across registry, cert, revocation, expiry, DNS, audit chain, and risk signals.

🔒

TLS 1.3 Simulation

Full handshake workflow with client/server random, pre-master secret derivation, cipher suite negotiation (AES-256-GCM-SHA384).

📡

DNSSEC Signing

CA-signed DNS records with RRSIG-style signatures. Independent verification detects tampered domain-to-IP mappings.

🔗

VPN Tunnel Simulation

Encrypted peer tunnels with AES-256-GCM + HMAC-SHA384. Only verified, non-revoked nodes can establish tunnels.

🗳️

Governance Voting

Decentralised proposal system with community voting. Proposals auto-resolve after 3 votes (simple majority). Full audit trail.

🚫

Revocation & Renewal

Compromised certificates are revoked (BLOCKED verdict). Renewed certificates preserve IPv6 binding with fresh validity.

📋

Signed Audit Chain

Append-only event log with prev_hash linkage, per-event SHA-256 hashing, and RSA signature. Tamper-evident.

🚨

Spoof Detection

Registry tampering is caught by CA signature verification. Attackers cannot forge trust without the CA private key.

🖥️

Triple Interface

CLI (full automation), Desktop GUI (tkinter), and Web Dashboard (React + Vite). All share the same engine.

🗺️

Network Topology

Live SVG force-directed graph showing nodes colored by trust posture with VPN/TLS connection edges.
